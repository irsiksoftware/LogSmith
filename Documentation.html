Introduction

LogSmith is a production-grade Unity logging package built from scratch to leverage Unity's native com.unity.logging system with advanced features for professional game development.

Designed for Unity 6000.2 with broad compatibility back to Unity 2022.3 LTS, LogSmith provides a clean, extensible architecture that grows with your project - from prototype to production.

Key Features

  • Native Unity Backend - Built on Unity's official logging system
  • DI-First Architecture - Full VContainer integration with no-DI fallback
  • Runtime Category Management - Add/remove categories at runtime via editor
  • Message Templating - Customizable output formats with live preview
  • In-Game Debug Overlay - Press F1 for real-time log filtering
  • 100% Test Coverage - EditMode + PlayMode tests with CI gates
  • Extensible Sinks - Console, File, and custom sink support
Open Source: LogSmith (Free) is open source under the MIT License. LogSmith Pro ($14.99) adds enterprise sink integrations (Sentry, Seq, Elasticsearch, HTTP).

Installation

Via Unity Package Manager (UPM)

Add LogSmith to your project using Unity's Package Manager:

  1. Open Window → Package Manager
  2. Click the + button
  3. Select Add package from git URL
  4. Enter: https://github.com/irsiksoftware/LogSmith.git
  5. Click Add

Via Package Manager (Asset Store)

Download LogSmith (Free) from the Unity Asset Store:

  1. Open Window → Package Manager
  2. Switch to My Assets
  3. Find LogSmith
  4. Click Download then Import

Requirements

  • Unity Version: 2022.3 LTS or later (tested on 2022.3, 2023, 6000.2)
  • Dependencies: com.unity.logging (automatically installed)
  • Optional: VContainer 1.17.0+ for DI features
Platform Support: LogSmith supports all Unity platforms. File sink features may be limited on WebGL and some console platforms due to filesystem restrictions.

Quick Start FREE

Option 1: Static API (No DI Required)

The simplest way to use LogSmith - just call the static API: 

using IrsikSoftware.LogSmith;

public class GameManager : MonoBehaviour
{
    private void Start()
    {
        // Get a logger for this category
        var log = LogSmith.GetLogger("Gameplay");

        // Log at different levels
        log.Info("Game starting");
        log.Warn("Low memory detected");
        log.Error("Failed to load level");
    }
}

Option 2: With VContainer

For projects using VContainer, LogSmith can be injected: 

using IrsikSoftware.LogSmith;
using VContainer;
using VContainer.Unity;

public class GameLifetimeScope : LifetimeScope
{
    protected override void Configure(IContainerBuilder builder)
    {
        // Register LogSmith (automatic)
        builder.RegisterLogSmith();

        // Your other registrations...
        builder.RegisterEntryPoint();
    }
}

public class GameManager : IStartable
{
    private readonly ILog _log;

    // Constructor injection
    public GameManager(ILog log)
    {
        _log = log;
    }

    public void Start()
    {
        _log.Info("Game manager started");
    }
}

Available Log Levels

  • Trace(string message) - Most verbose, for detailed debugging
  • Debug(string message) - General debugging information
  • Info(string message) - Informational messages
  • Warn(string message) - Warning messages
  • Error(string message) - Error messages
  • Critical(string message) - Critical failures

Configuration FREE

Creating LoggingSettings

Create a LoggingSettings asset to configure LogSmith:

  1. Right-click in Project window
  2. Select Create → LogSmith → Logging Settings
  3. Name it (e.g., "LoggingSettings")
  4. Configure via the editor window

Editor Window

Open the LogSmith settings editor:

  1. Go to Window → LogSmith → Settings
  2. Configure tabs:
    • Categories - Add/edit categories, set minimum levels, assign colors
    • Sinks - Enable Console/File sinks, configure paths
    • Templates - Customize message formats with live preview

In-Game Debug Overlay

Press F1 in Play mode to toggle the debug overlay. Features:

  • Filter by log level (Trace, Debug, Info, Warn, Error, Critical)
  • Filter by category
  • Search text filtering
  • Color-coded categories for easy identification
  • Minimal performance impact
Tip: The debug overlay works across all render pipelines (Built-in, URP, HDRP) and automatically adapts to your project's configuration.

VContainer Integration FREE

Setup with VContainer

LogSmith provides first-class VContainer support with automatic registration: 

using IrsikSoftware.LogSmith.DI;
using VContainer;
using VContainer.Unity;

public class GameLifetimeScope : LifetimeScope
{
    [SerializeField] private LoggingSettings settings;

    protected override void Configure(IContainerBuilder builder)
    {
        // Register LogSmith with custom settings
        builder.AddLogSmithLogging(settings);

        // Or use default settings
        // builder.AddLogSmithLogging();

        // Register your game systems
        builder.RegisterEntryPoint();
        builder.Register(Lifetime.Singleton);
    }
}

Constructor Injection

Inject ILog into any class registered with VContainer: 

public class PlayerController
{
    private readonly ILog _log;

    public PlayerController(ILog log)
    {
        _log = log;
    }

    public void TakeDamage(int amount)
    {
        _log.Warn($"Player took {amount} damage");
    }
}

Category-Specific Loggers

Create category-specific loggers for better organization: 

public class NetworkManager
{
    private readonly ILog _log;

    public NetworkManager(ILog log)
    {
        // Create a category-specific logger
        _log = log.WithCategory("Network");
    }

    public void Connect()
    {
        _log.Info("Connecting to server...");
    }
}
No DI? No Problem: LogSmith automatically falls back to a static API if VContainer isn't available. The same code works either way!

Sinks

Built-in Sinks FREE

LogSmith includes two production-ready sinks:

  • ConsoleSink - Outputs to Unity Console with color support
  • FileSink - Writes logs to text files with rotation support

Pro Sinks PRO $14.99

LogSmith Pro adds enterprise-grade integrations:

  • HttpSink - Send logs to any HTTP/REST endpoint
  • SentrySink - Automatic error tracking with Sentry
  • SeqSink - Structured logging with Seq (CLEF format)
  • ElasticsearchSink - Bulk indexing to Elasticsearch (ECS compatible)

Custom Sinks

Create your own sinks by implementing ILogSink: 

using IrsikSoftware.LogSmith;

public class CustomSink : ILogSink
{
    public string Name => "Custom";

    public void Write(LogMessage message)
    {
        // Your custom logging logic
        Debug.Log($"[{message.Level}] {message.Message}");
    }

    public void Flush()
    {
        // Flush any buffered messages
    }
}

// Register your sink
LogSmith.Router.RegisterSink(new CustomSink());

API Reference

ILog Interface

public interface ILog
{
    void Trace(string message);
    void Debug(string message);
    void Info(string message);
    void Warn(string message);
    void Error(string message);
    void Critical(string message);
    ILog WithCategory(string category);
}

Static API

// Get default logger
var logger = LogSmith.Logger;

// Create category-specific logger
var gameplayLog = LogSmith.GetLogger("Gameplay");

// Access router for advanced configuration
var router = LogSmith.Router;

// Reload settings at runtime
LogSmith.ReloadSettings();

// Switch output format
LogSmith.SwitchFormat(MessageFormat.JSON);

LogRouter

// Register custom sinks
router.RegisterSink(new CustomSink());

// Unregister sinks
router.UnregisterSink(sink);

// Set global minimum level
router.SetGlobalMinimumLevel(LogLevel.Info);

// Set per-category minimum level
router.SetCategoryFilter("Network", LogLevel.Debug);

Troubleshooting

Logs Not Appearing

  • Check that the category is enabled in the LogSmith settings
  • Verify the minimum log level allows your message through
  • Ensure Console/File sinks are enabled in settings
  • Check Unity's Console window filters aren't hiding messages

VContainer Not Working

  • Ensure VContainer 1.17.0+ is installed
  • Check that builder.AddLogSmithLogging() is called
  • Verify your LifetimeScope is active in the scene
  • LogSmith will fall back to static API if VContainer isn't found

File Sink Not Writing

  • Check file path permissions
  • Verify platform supports file I/O (WebGL limitations)
  • Check logs directory exists or can be created
  • Look for errors in Unity Console about file access

Performance Issues

  • Disable Trace/Debug levels in production builds
  • Use category filtering to reduce log volume
  • Disable debug overlay in builds (it's editor-only by default)
  • Check sink configurations for batching/buffering options
Platform-Specific Issues: Some platforms (WebGL, iOS, consoles) have restrictions on file I/O and threading. LogSmith detects these at runtime and adapts gracefully.
